# Complete the LLM/TTS fallback (step 7) + v1.1 vision (step 8)

## Context

The corpus → review → push loop is fully built (steps 1–6). Two pieces from
[docs/specs/implementation_plan_v1.md](../docs/specs/implementation_plan_v1.md)
remain deferred per
[docs/status/implementation-status.md](../docs/status/implementation-status.md):

1. **Step 7 — LLM/TTS fallback.** Today [pipeline/select.py](../src/anki_builder/pipeline/select.py)
   routes a word to `needs_fallback` when no Tatoeba candidate survives the
   audio+translation filter, and [cli.py](../src/anki_builder/cli.py)`::cmd_run`
   only *marks* it (`status=needs_fallback`, `flag=fallback`, no fields). The
   `[llm]`/`[tts]` config is parsed but unused; no `llm/` package exists; nothing
   loads `OPENAI_API_KEY` from `.env`.
2. **Step 8 — v1.1 vision.** No image path at all; `cli.py` only accepts
   `--word`/`--words`.

**Goal:** make v1.1 complete — a real OpenAI-SDK fallback that generates a
flagged, review-gated card with TTS audio, and a functional vision step that
extracts words from an image into the same pipeline.

**Confirmed decisions (this session):**
- Vision is **fully functional** (`run --image`), not a stub.
- Fallback runs **during `run`** (plan-aligned), gated by
  `cfg.llm.fallback_enabled` AND a present `OPENAI_API_KEY`; the card is
  enqueued `status=pending` so it still passes the mandatory D2 review gate.

The whole point is to preserve the existing seams: fallback reuses the
`needs_fallback` branch; vision feeds plain word strings into the unchanged
per-word pipeline ("nothing downstream changes").

## Approach

### A. Dependencies & config (`pyproject.toml`, `config.py`)

- Add `openai>=1.0` and `python-dotenv>=1.0` to `[project].dependencies`.
- In [config.py](../src/anki_builder/config.py) `load_config`, call
  `load_dotenv()` (best-effort, guarded import) *before* reading
  `os.environ["OPENAI_API_KEY"]`, so `.env` is honored (closes the CLAUDE.md
  "nothing loads .env yet" gap). `LLMConfig.api_key`/`base_url` and `TTSConfig`
  already exist — no schema change.

### B. The OpenAI client seams (`llm/` — new package)

Mirror the project's single-wire-seam convention (`AnkiClient.invoke`): one tiny
method per file does the network call so tests monkeypatch exactly that. Import
`openai` lazily inside those methods (keeps mere import cheap and offline).

- **`llm/client.py`** — `LLMClient(cfg.llm)`.
  - `_chat(messages, *, json=True) -> str` — the only LLM wire seam; builds
    `OpenAI(api_key, base_url)` and calls `chat.completions.create(model=…,
    response_format={"type":"json_object"})`, returns the message content.
  - `generate_sentence(word, *, target_lang, base_lang) -> FallbackSentence`
    (dataclass: `spa_text`, `translation`, `gloss`) — prompts for a short, simple
    target-language sentence using `word`, its base-language translation, and a
    short L1 gloss; parses the JSON.
  - `extract_words(image_b64, mime) -> list[str]` — vision call via the same
    `chat.completions` shape with an `image_url` data-URL part; prompts to return
    `{"words":[...]}` (lemmas; only marked/underlined ones if any are marked).
    Reuses the configured vision-capable model (D4).
- **`llm/tts.py`** — `TTSClient(cfg.tts, *, api_key, base_url)` (reuses the LLM
  key/base_url per D4).
  - `_speech(text) -> bytes` — the only TTS wire seam; `audio.speech.create(
    model=cfg.tts.model, voice=cfg.tts.voice, input=text, response_format="mp3")`.
  - `synthesize_to(text, dest: Path) -> Path` — writes bytes atomically
    (`.part` → replace, like [tatoeba/audio.py](../src/anki_builder/tatoeba/audio.py)`::_stream_to`).
- **`llm/vision.py`** — `extract_words(image_path, *, cfg, client=None) ->
  list[str]`: read file, sniff mime from suffix, base64-encode, delegate to
  `LLMClient.extract_words`. Thin wrapper so the CLI imports one symbol.

### C. Fallback card assembly (`pipeline/fallback.py` — new, + `pipeline/cards.py`)

- Add `cards.build_fallback_fields(word, spa_text, translation, *,
  word_translation, audio_filename) -> CardFields` — reuses `blank_sentence` and
  sets `audio=f"[sound:{audio_filename}]"` (or `""`), `source="LLM fallback"`,
  `flag="fallback"`. (A synthetic `Candidate` won't work because `sound_tag`
  needs a real `sentence_id`; a dedicated builder keeps cards.py the one place
  fields are assembled.)
- **`pipeline/fallback.py`** — `generate_fallback(word, *, cfg, conn,
  llm_client=None, tts_client=None) -> FallbackResult(fields, audio_filename)`:
  1. `fb = llm.generate_sentence(word, …)`.
  2. `gloss = fb.gloss or queries.gloss_for(conn, word)` (prefer LLM gloss, fall
     back to the FreeDict dict).
  3. `audio_filename = audio.fallback_media_filename(word, target_lang)`; try
     `tts.synthesize_to(fb.spa_text, media_cache/audio_filename)`. On TTS failure,
     set `audio_filename=None` (card pushed silent but still flagged) — never
     abort.
  4. `fields = cards.build_fallback_fields(…)`; return both.

### D. Audio filename + push/serve path for TTS files

- **`tatoeba/audio.py`** — add `fallback_media_filename(word, lang) ->
  f"fallback_{lang}_{slug}.mp3"`, where `slug` is a filesystem-safe slug of the
  word with a short hash suffix to stay unique/valid for any input.
- **`anki/push.py`** — extract a public `media_path_for_row(row, cfg) ->
  Path|None`: if `chosen_sentence_id` → `download_audio(...)` (today's Tatoeba
  path); elif `audio_filename` and the cached file exists → that path (the
  TTS file); else `None`. Use it inside `push_accepted` (replacing the inline
  `sid and fname` block) so fallback rows store their cached mp3 + `addNote`
  normally.

### E. Wiring `run` (`cli.py`)

- In `cmd_run`'s `selection.needs_fallback` branch:
  - `--dry-run` → print the deferred line, **no network** (unchanged).
  - else if `cfg.llm.fallback_enabled and cfg.llm.api_key` → `try:
    fallback.generate_fallback(...)` then `queries.enqueue(status="pending",
    chosen_sentence_id=None, candidates=None, fields=…, audio_filename=…,
    flag="fallback")`; print "LLM fallback generated (flagged, pending review)".
    On exception, warn and fall through to the marker below.
  - else → today's marked-only `needs_fallback` enqueue, with the message
    nudging to set `OPENAI_API_KEY` / `fallback_enabled`.
  - Add a `--no-fallback` flag to `run` to force the offline marker path (handy
    for tests / offline mining).
- Add `--image PATH` to the `run` mutually-exclusive group; `_load_words` gains:
  `if args.image: return vision.extract_words(Path(args.image), cfg=cfg)`. The
  rest of `cmd_run` is untouched — each extracted word flows through the normal
  search→rank→select (→ fallback) path.

### F. Review app: serve + show fallback audio

- **`review/server.py`** `get_audio`: replace the `chosen_sentence_id`-only
  logic with `anki_push.media_path_for_row(row, cfg)` so fallback TTS files are
  served (it already calls into `anki_push`).
- **`review/static/index.html`**: render the `<audio>` player when
  `row.audio_filename` is present (not only `row.chosen_sentence_id`), so a
  reviewer can hear the TTS clip. The `Flag` badge already renders.

### Critical files

- New: `src/anki_builder/llm/__init__.py`, `llm/client.py`, `llm/tts.py`,
  `llm/vision.py`, `pipeline/fallback.py`,
  `tests/test_fallback.py`, `tests/test_vision.py`.
- Edited: `pyproject.toml`, `config.py`, `pipeline/cards.py`, `tatoeba/audio.py`,
  `anki/push.py`, `cli.py`, `review/server.py`, `review/static/index.html`.
- Reuse: `blank_sentence`/`build_source` ([pipeline/cards.py](../src/anki_builder/pipeline/cards.py)),
  `gloss_for`/`enqueue` ([db/queries.py](../src/anki_builder/db/queries.py)),

  `note_payload`/`Flag` badge ([anki/model.py](../src/anki_builder/anki/model.py)).
